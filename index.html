ClickUp PMO Workflow - AI-Driven Intake to Delivery

Design & Pilot
Architect / Builder · ACRRM - Digital Technology & Innovation

Designed a full PMO delivery framework that takes requests from a single AI intake point, auto-classifies them by complexity, and routes them through governance-tiered delivery paths - each with AI-generated BA and PM artifacts. BAU goes straight through. Config Items get light governance. Projects get the full treatment.

ClickUp n8n Cloud Claude API ClickUp Brain AI Artifact Gen
▼ View detail

Problem

Traditional PMO workflows treat every request the same - either everything gets full project governance (slow and expensive) or nothing does (chaotic and untracked). There's no intelligent middle ground. Teams waste time producing artifacts for work that doesn't need them, while complex projects skip governance because the overhead feels disproportionate.

Delivery Framework

Three Delivery Tiers

  • BAU Flow - Straight-through processing. Auto-assign, develop, quick test, deploy, auto-close. SLA: 24-48hrs. No artifacts, no overhead.
  • CI Flow - Light governance. AI generates Requirements Doc, User Stories, Acceptance Criteria, Impact Analysis. BA reviews and approves. Sprint cycles with auto-updated PMO reporting. 1-2 sprints.
  • Project Flow - Full governance. AI generates both BA artifacts (Requirements, User Stories, Process Flows, Acceptance Criteria) and PM artifacts (Project Charter, Plan, RACI Matrix, Risk Register). Human review gates. UAT, go-live, AI-generated lessons learned.

Key Innovation

AI agents draft the artifacts, humans review and approve them. This inverts the traditional model where humans write everything from scratch. The BA and PM still own quality and decisions - but they start from a structured draft instead of a blank page. Automated PMO dashboards track velocity, milestones, budget, and risk without manual status updates.

Open Brain - Cross-Tool AI Memory System

Active
Builder / Architect · Personal / ACRRM

A shared knowledge system that gives every AI tool persistent memory. Built on pgvector embeddings and MCP, Open Brain lets Claude, Gemini, and other AI assistants share context, remember decisions, and pick up where any tool left off - eliminating the “cold start” problem of AI conversations.

MCP n8n Cloud PostgreSQL / pgvector Node.js Railway OpenAI Embeddings
▼ View detail

Problem

Every AI conversation starts from zero. Context from a Claude session doesn’t carry to the next one, let alone to Gemini or ChatGPT. Decisions get re-explained, project context gets lost, and working across multiple AI tools means constant repetition. There’s no shared memory layer.

Architecture

AI Tool (Claude/Gemini/ChatGPT) → MCP Protocol → Open Brain Server (Node.js) → n8n Workflows → Neon PostgreSQL + pgvector

Key Components

  • Vector Knowledge Base - Neon PostgreSQL with pgvector extension. Every piece of saved knowledge gets embedded (OpenAI text-embedding-3-small, 1536 dimensions) for semantic search
  • MCP Server - Custom Node.js server exposing tools: search_knowledge, save_memory, log_conversation, get_user_context, get_project_context. Supports stdio (local) and SSE (remote) transport
  • n8n Orchestration - Two core workflows: Document Ingestion (chunk → embed → store) and Semantic Search (embed query → vector similarity → return ranked results)
  • Railway Deployment - SSE server deployed for remote AI tool access. Claude Desktop uses stdio for low-latency local connection
  • Cross-Tool Memory - Conversation logs, project decisions, technical patterns, and domain knowledge persist across all connected AI tools

Outcome

Claude now has persistent project context across sessions - it knows the tech stack, past decisions, architectural patterns, and current status without re-explanation. The same knowledge base is accessible from any MCP-compatible AI tool. Currently the primary knowledge store for all automation project context at ACRRM.

Operating Principles

01

Intake Funnel

Every AI initiative enters through a single intake process. No shadow projects. Requests triaged, scoped, and assigned a tier within 48 hours.

02

2-Week Sprints

All active work runs in agile two-week cycles with demos and retros. Budget caps enforced per tier to prevent scope creep.

03

Chargeback Model

Costs attributed to requesting business units. Creates natural prioritisation pressure - only high-value work progresses past Explore.

04

Model Agnostic

Back the field, not the horse. No vendor lock-in. Solutions designed to swap underlying models as the landscape evolves.

05

Change + Delivery

Integrate with change management and project teams to ensure AI initiatives land smoothly - not just technically, but organisationally.

06

HR + L&D Partnership

Partner with HR and Learning & Development to assess staff impact, upskill teams, and manage the human side of AI-driven transformation.

07

Data Privacy + Sovereignty

All data handling aligns with the Privacy Act and APPs. Health data classified per OAIC guidelines. Sovereign data stays sovereign.

08

Risk + Ethics Assessment

Algorithmic impact assessments at the Validate gate. Bias testing, fairness criteria, and ethical review before any model touches real data.

09

Continuous Evaluation

Live AI systems undergo periodic review. Model drift, performance degradation, and revalidation on a defined cadence. Governance doesn't stop at go-live.
